# Aerodynamics4MC Integration Notes

## Current Boundary

Aerodynamics4MC is useful to FPV Dronecraft as a world wind provider. Its public API exposes trusted gameplay wind samples with mean wind, bounded gusts, pressure anomaly/proxy, turbulence, shear, shelter, confidence, humidity, temperature, and ABL stability/mixing diagnostics. It does not replace FPV Dronecraft's multirotor model: rotor thrust, induced flow, vortex ring state, propwash, battery, motor dynamics, and airframe forces still live in `drone-sim-core`.

The first integration keeps that boundary explicit:

- `fabric-mod` samples Aerodynamics4MC only when the mod is present at runtime.
- The bridge uses `GameplayWindSample` with `GAMEPLAY_SERVER_ONLY`, so server physics ignores client-local L2 wind.
- Missing or incompatible Aerodynamics4MC falls back to FPV Dronecraft's existing Minecraft-weather wind.
- `/fpvdrone environment wind ...` remains the highest-priority override.
- The simulation core stays Minecraft-free and does not depend on Aerodynamics4MC classes.

## Upstream API Audit

The current Aerodynamics4MC Core API keeps the server gameplay entry point compatible with the bridge: `AeroMinecraftWindApi.sampleGameplay(ServerLevel, Vec3, SamplePolicy)` returns `GameplayWindSample`, and the gameplay default is still `GAMEPLAY_SERVER_ONLY`. FPV Dronecraft deliberately passes that policy explicitly so authoritative physics continues to ignore client-local/debug flow.

The bridge now tracks the full gameplay sample contract used by the simulator: mean, effective, and explicit gust vectors; pressure; turbulence; shear; shelter; updraft; source temperature and humidity; confidence; source level; authority; L1/world-delta/L2 freshness epochs; and ABL stability/mixing diagnostics. Optional reflection is kept for fields that are useful telemetry or local L2 modifiers, so an older A4MC build can degrade to unavailable values without disabling the whole wind bridge.

Aerodynamics4MC also exposes lower-level L2 request/result types with grid dimensions, cell size, time step, inlet velocity, density, viscosity, solid masks, optional flow-atlas output, and optional force/moment output through `AeroL2ForceMoment`. Treat that as the next bounded CFD refinement path, not as part of the gameplay wind bridge. A future integration should run L2 force/moment requests behind availability and latency budgets, most likely for offline calibration packets or a separate server-authoritative local-flow solver, then feed only capped force, moment, pressure-center, or voxel-flow summaries into `drone-sim-core`.

## Implemented Path

`Aerodynamics4McWindBridge` reflects against `AeroMinecraftWindApi.sampleGameplay(ServerLevel, Vec3, SamplePolicy)`. It binds A4MC's explicit `gustVelocityVector()` when available and falls back to `effectiveVelocityVector() - meanVelocityVector()` for older bridge-compatible APIs. A sample with flow is preserved for telemetry, while trusted/confident/fresh samples feed `DroneEnvironment` as natural wind, including the current stage-one playable environment when Aerodynamics4MC is installed:

- mean wind becomes the base wind vector after confidence/freshness weighting against Minecraft weather or calm fallback, while explicit gust remains separated for transient source-gust adoption,
- API turbulence becomes the natural turbulence floor and barometer sensor-noise input, with the core falling back to quality-weighted raw A4MC source turbulence when direct callers provide raw telemetry before the fabric bridge has folded it into environment turbulence,
- shear and shelter add bounded dirty-air turbulence,
- trusted local L2 pressure anomaly/proxy adds a small capped turbulence boost, so block-edge or duct pressure disturbance can excite rotor response without turning coarse weather pressure into local CFD forcing,
- A4MC shear/updraft/shelter diagnostics add a bounded terrain-shear gust component to the effective air mass, with ABL stability/mixing shaping its response and decay without changing the bounded target magnitude,
- trusted A4MC updraft now also becomes its own smoothed vertical air-mass split before terrain-shear synthesis, subtracting explicit A4MC vertical gust and already adopted vertical mean wind before adoption so effective-flow `Y` is not counted twice, with ABL stability/mixing shaping adoption so mixed unstable thermals respond more strongly while stable low-mixing layers remain damped,
- gust vector, with gust-speed fallback for older bridges, adds a bounded unresolved turbulence boost plus a small coherent A4MC source-gust air-mass component whose vertical and scalar response is shaped by ABL stability/mixing; the turbulence/updraft contribution also removes explicit `gustY` and represented vertical mean wind first so vertical gust energy is not counted as both gust and updraft,
- raw API temperature and humidity are preserved for telemetry; the core applies confidence/freshness weighting to source temperature before deriving effective air temperature for density, Mach/Reynolds, icing, barometer, and thermal models, while humidity is quality-weighted inside the core before contributing to moist-air density relief,
- raw pressure anomaly is preserved for telemetry, then confidence/freshness weighted before adjusting effective air-density ratio and barometer pressure, and trusted local L2 pressure also contributes a small bounded barometer static-port bias while staying bounded for L2 proxy values,
- source level, authority, and freshness age are preserved for L0/L1/L2 regression analysis, with stale source samples faded out of base wind, atmospheric scalar, local voxel, disk-gradient, and turbulence coupling,
- raw A4MC mean/effective/gust/turbulence split is preserved alongside confidence/freshness/source-quality telemetry, while adopted base wind and core transients use the shared quality gate,
- quality-weighted A4MC humidity participates in moist-air density, bounded moist-air dynamic viscosity for rotor Reynolds/low-Re effects, thermal ventilation cooling, and low-temperature saturated-air icing, so hot humid air can change rotor authority/Reynolds behavior and slow motor/ESC/battery heat rejection while freezing fog-like humidity can add light time-accumulating rotor icing without being treated as rain film,
- trusted local L2 A4MC shelter reduces the direct precipitation exposure used by body and rotor rain-film sampling, so covered tunnel mouths and wall shadows shed some wet-prop load without changing Minecraft's rain state or water-immersion model,
- asymmetric trusted local L2 A4MC shelter/obstacle residuals and radially aligned rotor pressure-gradient wind equivalents now add a small source-availability-gated airframe pressure-center migration from adopted per-rotor signals, letting block-edge or tunnel-mouth shielding create body-scale aerodynamic moments during cruise or powered hover instead of only rotor-local obstruction,
- the core flight model now applies the same trust/confidence/freshness quality gate to A4MC source-gust, terrain-shear target/decay, and local-voxel ventilation transients, so direct `DroneEnvironment` callers cannot excite or reshape those paths with stale or untrusted samples,
- untrusted A4MC flow samples remain visible as zero-quality wind-source telemetry instead of being collapsed into `unavailable`, making client-local or otherwise non-authoritative sources auditable without letting them affect server physics,
- ABL stability and mixing modulate the core Dryden turbulence intensity/correlation time and the near-surface boundary-layer wind profile: unstable mixed layers strengthen and quicken gust response while ventilating ground/ceiling shear faster, and stable low-mixing layers damp Dryden response while leaving stronger surface shear and recirculation.

When trusted Aerodynamics4MC samples are available, `fabric-mod` samples each rotor center plus weighted disk-edge points and passes both disk-mean wind and a bounded in-plane axial wind gradient into `drone-sim-core`. The disk-edge directions are projected onto each rotor's actual thrust-axis plane before world sampling, so outward-canted or otherwise tilted rotors read local voxel flow from their real disk plane instead of a fixed body X/Z plane. The bridge explicitly requests A4MC `SamplePolicy.GAMEPLAY_SERVER_ONLY`, so server-trusted L2 can refine block-edge airflow while client-local/debug L2 remains excluded from authoritative flight physics. Missing or low-quality disk-edge samples remain in the disk quadrature as center-wind fallback, with confidence/freshness fading applied before the disk blend, making partial local voxel coverage conservative instead of amplifying one-sided samples into a full-disk gradient. The body-center source-quality check only decides whether rotor-field sampling is available; disk/local-voxel outputs keep the per-sample quality already embedded in their blends instead of multiplying by body quality again. The same disk-edge samples also calculate bounded shelter and pressure gradients: uneven A4MC shelter around a rotor disk contributes a small directional side-flow obstruction hint, while local L2 pressure differences are converted into a small equivalent disk-wind gradient that reuses the existing asymmetric rotor-loading path. Missing, stale, or coarse shelter/pressure samples fade back to the rotor-center value instead of creating a false wall direction. The separated per-rotor pressure-gradient wind-equivalent array is also treated as an adopted local-voxel signal in the core, with source freshness/trust used only to reject unavailable local flow rather than attenuating that array a second time. The rotor disk wind blend uses A4MC mean wind only, so coherent source gust remains separated for the core air-mass transient instead of being folded into both disk gradient and source-gust response. The core also uses the adopted per-rotor local-voxel shelter/residual and pressure-gradient imbalance to add a bounded airframe pressure-center offset, but the pressure-gradient portion is first projected onto each rotor arm's radial body direction; one-sided block shielding or radially aligned pressure steps can therefore create a small body-scale moment in straight cruise without being squared down by body-center confidence, while purely tangential rotor-disk pressure gradients stay rotor-local instead of moving the whole airframe pressure center. The fabric coupling adopts the quality-weighted A4MC mean wind as the base environment wind and leaves explicit gust energy for source-gust transients; it passes Minecraft-derived ambient temperature plus raw A4MC source temperature separately so the core owns the single quality-gated temperature adoption used by density, rotor aerodynamics, icing, barometer, and thermal systems. It also de-overlaps explicit vertical gust and the already adopted vertical mean wind before adding updraft-derived natural turbulence, so the environment turbulence floor mirrors the core air-mass split instead of double-counting A4MC effective-flow `Y`. For direct core callers, the same floor is derived from the greater of explicit environment turbulence and quality-weighted raw A4MC source turbulence, so raw telemetry does not need a fabric-only pre-folding step to drive Dryden-scale response or barometer sensor-noise amplitude. The core keeps its existing global Dryden and dirty-air wind model, folds the quality-weighted A4MC gust vector into a small coherent source-gust air-mass vector, shapes source-gust vertical mixing with ABL diagnostics, adopts trusted A4MC updraft as a separate smoothed vertical air-mass vector after removing explicit vertical gust already represented by the source-gust split and vertical mean flow already represented by the base wind, then applies the per-rotor wind delta to axial gust and lateral inflow calculations. ABL stability/mixing now also gates that updraft target and smoothing time: mixed unstable layers strengthen and quicken thermals or shaft-like rising flow, while stable low-mixing layers damp and slow the vertical split. The same ABL gate now shapes ground/ceiling boundary-layer wind recovery and shear-layer dirty-air/recirculation: mixed unstable layers recover horizontal wind toward freestream and shed less surface shear, while stable low-mixing layers preserve a stronger surface wind deficit and recirculation penalty. When only the gust speed scalar is available, the core preserves the earlier bounded synthetic gust direction but applies the same ABL scalar/vertical shaping. A4MC shear/updraft diagnostics still drive a small correlated terrain-shear wind component in the core air-mass model, with A4MC shelter increasing the wake-like component only when mean wind is present; ABL stability and mixing now tune that component's smoothing time so mixed unstable layers flush faster while stable low-mixing layers persist longer, and that terrain-shear path uses the same de-overlapped updraft signal. This lets cliffs, openings, and sheltered voxels produce time-varying relative-air changes even when the mean source wind is steady. A4MC humidity now feeds the core moist-air model beyond density by applying a bounded vapor-pressure-based dynamic-viscosity relief to rotor Reynolds/low-Re calculations, reducing the dry-air thermal ventilation term for motors, ESCs, and the battery pack under hot humid conditions, and contributing a bounded freezing-humidity equivalent wetness to the time-accumulating icing model when the air is both cold and near saturated. The core derives adopted temperature and humidity through the same trust/confidence/freshness gate used by A4MC transients, so stale or untrusted atmospheric scalars remain logged but cannot change density, rotor Reynolds behavior, cooling, icing, barometer pressure, or sensor turbulence noise. That freezing humidity path does not wet the prop film or trigger rain-film thrust loss, leaving water immersion and precipitation film cooling separate. The disk gradient adds first-order asymmetric disk loading: small thrust loss, extra aerodynamic load/vibration, bounded local dynamic-stall onset, and flapping tilt toward the higher axial-flow side. This keeps Minecraft-specific wind sampling outside the physics core while allowing rotor-to-rotor and disk-scale flow differences near cliffs, openings, and sheltered blocks.

Powered hover now opens that same local pressure-center path through a small rotor-wash downforce gate. A trusted asymmetric local L2 offset can therefore create a bounded body-scale moment beside blocks or tunnel mouths even when freestream airspeed is near zero, while stale, coarse, or untrusted samples still fade to no body moment.

`DroneEnvironment` now carries wind-source telemetry through to the blackbox CSV and summary digest. The log records whether the frame used calm, Minecraft weather, an explicit environment override, or Aerodynamics4MC, plus source confidence, raw source turbulence, source-quality factor, trusted state, source level, authority, freshness age, raw mean/effective/gust speed split, raw source gust vector, raw pressure anomaly, raw shear/shelter/updraft, local voxel-flow availability, raw source temperature, raw source humidity, effective ambient air temperature, raw ABL stability/mixing, per-rotor disk-gradient magnitudes, the separated A4MC pressure-gradient wind-equivalent contribution inside that disk gradient, disk-gradient thrust-loss/load/vibration/stall response components, per-rotor local voxel obstacle residuals, A4MC shelter-gradient obstruction magnitudes, rotor-flow obstruction wall-force geometry factors, aggregate/per-rotor A4MC ventilation efficiency for motor/ESC cooling, pack-level A4MC ventilation efficiency for battery cooling, airframe pressure-center torque axes, per-rotor ground/ceiling supported-surface coverage and gate values, the adopted A4MC source-gust split including its world-vector components, the adopted signed A4MC updraft split, and the A4MC terrain-shear gust split. The summary reports A4MC sample count, trusted/untrusted/local-L2 counts, L0/L1/L2 counts, stale/freshness age, source mean/effective/gust peaks, peak confidence, peak source turbulence, peak source quality, pressure anomaly, shelter, source shear, raw updraft, ABL stability/mixing, disk-gradient magnitude, peak A4MC pressure-gradient wind, peak airframe pressure-center torque, minimum local voxel obstacle residual, peak A4MC shelter-gradient obstruction, peak rotor-flow obstruction wall-force factor, minimum rotor/pack A4MC ventilation efficiency, peak disk-gradient thrust-loss/load/vibration/stall response, peak barometer pressure-port error, per-rotor precipitation wetness range for valid airframe rotors, peak A4MC source gust, peak adopted A4MC updraft, and peak A4MC terrain-shear gust, making later terrain-flow tuning auditable instead of relying on inferred wind state.

In the advanced environment path, local obstacle scans, drone wake, water, precipitation, ground effect, ceiling effect, and rotor disk obstruction remain active on top of the sampled wind. If Aerodynamics4MC reports trusted local L2 voxel flow, FPV Dronecraft treats A4MC shelter as having already explained part of the wall/tunnel wind shadow and attenuates only the duplicated local obstacle airflow plus rotor side-flow obstruction. Rotor-center A4MC samples now refine that attenuation per rotor, so the wall-side rotor at a tunnel mouth can shed more duplicated Minecraft obstruction than the exposed rotor instead of inheriting only the body-center shelter residual. Rotor-center samples that are present but stale, coarse, or otherwise zero-quality keep the body residual fallback instead of restoring duplicate geometric obstruction abruptly. The same trusted local shelter now also attenuates precipitation exposure before the wet-prop model sees it: body-center shelter gates the body rain sample, while rotor-disk mean shelter gates each rotor's rain-film input, and no trusted local voxel sample leaves the existing Minecraft sky/rain exposure unchanged. Rotor rain and water-immersion disk samples now use the same per-rotor thrust-axis and disk-plane directions as A4MC and near-surface sampling, so canted rotors do not test wetness on a fixed body X/Z plane. Rotor-disk ground/ceiling samples also gate near-field lift by supported patch coverage, so partial blocks, holes, and ledges do not act like infinite plates. The same trusted local shelter now reduces motor and ESC ventilation cooling per rotor by combining raw body shelter, rotor-center local voxel residual, and disk shelter-gradient obstruction, while the battery pack receives a milder whole-frame ventilation penalty from the averaged rotor local-voxel samples. The core still quality-gates raw body shelter, while per-rotor residual and shelter-obstruction arrays carry the confidence/freshness weighting already applied by the fabric local-voxel bridge, so half-trusted A4MC local flow fades once instead of being squared down. Rotor ground effect, ceiling effect, wet props, precipitation, and drone wake stay independent. In the stage-one playable path, the environment remains calm unless Aerodynamics4MC or an explicit `/fpvdrone environment` override provides wind or turbulence.

The offline `wall_skim` trace now carries a synthetic trusted A4MC L2 local-voxel profile. It attenuates duplicated geometric wall obstruction with a local obstacle residual, injects per-rotor A4MC shelter-gradient obstruction and pressure-gradient-equivalent disk wind biased toward the wall-side rotors, applies a rainy local-shelter pass that lowers each valid rotor's precipitation wetness below the body rain value according to its synthetic A4MC shelter, and records the explicit source-quality factor, local voxel obstacle residual, A4MC shelter obstruction, separated A4MC pressure-gradient wind, total disk-gradient magnitude, combined rotor flow obstruction, body precipitation wetness, per-rotor precipitation wetness, pack ventilation efficiency, and aggregate/per-rotor rotor ventilation efficiency. This gives the CSV regression suite a stable rainy block-edge/tunnel-mouth proxy before live Aerodynamics4MC worlds are available in automated tests.

The offline recorder `Summary:` line now also prints `max_pc_torque`, `max_wallfac`, and `min_a4mc_vent=rotor/pack`, letting quick CLI runs audit the same pressure-center moment peak, rotor wall-force geometry gate, and minimum A4MC ventilation-efficiency loss that the CSV columns preserve sample by sample.
